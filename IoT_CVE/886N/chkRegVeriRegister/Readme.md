# TP-LINK WR-886N Vulnerability
This vulnerability is on the /cloud_config/router_post/check_reg_verify_code interface and affects the latest version of the router operating system (which is a different RTOS system from the linux operating system), and the firmware version is TL-WR886N V6.0 upgrade software 20190826 2.3.8
## Vulnerability description
There is a buffer overflow vulnerability on the /cloud_config/router_post/check_reg_verify_code interface.

1. An interface ```/cloud_config/router_post/check_reg_verify_code``` is registered first in the chkRegVeriRegister function.
![](./chkRegVeriRegister1.png)
2. In the handler, the v7 variable is the ```username``` field in the packet, and the v9 variable is the ```verify_code``` field in the packet. Then directly memcpy to the a3 variable refers to the address space, there is a buffer overflow problem.
![](./chkRegVeriRegister2.png)
